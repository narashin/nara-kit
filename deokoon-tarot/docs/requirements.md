## User Stories

### Cross-cutting (앱 골격)
- **US-1**: As 팬, 첫 진입 시 최애를 자유 텍스트로 입력해서, 이후 모든 질문이 그 최애를 맥락으로 해석되도록 한다.
- **US-2**: As 팬, 카드를 직접 셔플·뽑는 인터랙션을 거쳐서, 의식 같은 몰입감으로 결과를 받는다.
- **US-3**: As 팬, 룰베이스 카드 의미 + LLM 톤이 입혀진 종합 해석을 봐서, "정성스러운" 답을 받는다.
- **US-4**: As 팬, 앱을 홈화면에 설치(PWA)해서, 브라우저 거치지 않고 빠르게 진입한다.
- **US-5**: As 팬, 오프라인 상태에서도 룰베이스 결과를 봐서, 인터넷 없어도 최소 카드 의미는 확인한다.
- **US-6**: As 팬, 과거 본 결과 히스토리를 다시 봐서, 시간이 흘러 어떻게 됐는지 회고한다.

### Category-specific (8 카테고리, 전부 MVP 포함)
- **US-7** (compat): As 팬, 궁합 카테고리에서 나↔최애 양방향 케미를 본 카드로 받아서, 둘 사이 에너지를 가늠한다.
- **US-8** (encounter): As 팬, 마주침 카테고리에서 시기·장소·가능성을 함의하는 카드를 받아서, 우연한 만남의 운을 본다.
- **US-9** (evangelism): As 팬, 영업 카테고리에서 "영업 대상 → 최애" 통할지 카드를 받아서, 영업 전략을 가늠한다.
- **US-10** (ticketing): As 팬, 티켓팅 운 카테고리에서 예매일·좌석·성공률 함의 카드를 받아서, 마음의 대비를 한다.
- **US-11** (stamina): As 팬, 덕질 지속력 카테고리에서 권태기·열정 흐름 카드를 받아서, 덕질 페이스를 점검한다.
- **US-12** (comeback): As 팬, 컴백/활동운 카테고리에서 최애의 차기 활동운 카드를 받아서, 응원할 시기를 본다.
- **US-13** (onboarding): As 팬, 입덕 적기 카테고리에서 새 최애 입덕 타이밍 카드를 받아서, 추가 덕질의 결심을 한다.
- **US-14** (free): As 팬, 자유 질문 카테고리에서 자연어로 질문해서, 가장 가까운 스프레드로 매핑된 카드 결과를 받는다.

### Auth & Quota (decisions에서 파생)
- **US-15**: As 팬, Google 계정으로 로그인해서, 멀티디바이스에서도 최애·히스토리를 이어 사용한다.
- **US-16**: As 팬, 하루 5회 무료 한도 안내를 받아서, 무분별한 결과 소비 없이 신중하게 본다.

## Acceptance Criteria

### AC-1 (US-1, Happy) — 최애 입력 onboarding
Given Google 로그인을 막 마친 신규 사용자가
When 자유 텍스트로 최애 이름 (1자 이상 [UNVERIFIED] 50자 이하)을 입력하고 확인을 누르면
Then 최애 이름이 Firestore 사용자 프로필에 저장되고 메인 화면으로 진입한다

### AC-2 (US-1, Edge) — 최애 변경
Given 이미 최애가 등록된 사용자가
When 설정에서 "최애 변경" 진입 후 새 이름을 저장하면
Then 이후 모든 해석이 새 이름 맥락으로 생성되고 기존 히스토리는 옛 이름으로 보존된다

### AC-3 (US-2, Happy) — 셔플·드로우 인터랙션
Given 사용자가 카테고리를 선택하고 카드 뽑기 준비가 된 상태에서
When 셔플 제스처(스와이프 또는 버튼)를 거쳐 스프레드별 카드 수만큼 뽑으면
Then 뽑힌 카드가 정·역방향 무작위로 노출되고 결과 화면으로 이동한다

### AC-4 (US-3, Happy) — 정성스러운 해석 출력 (온라인)
Given 카드가 뽑힌 직후 (온라인 상태)
When 룰베이스 카드 의미가 호출된 뒤 LLM이 톤을 입히면
Then 결과 화면에 카드별 의미 요약과 최애 이름이 자연스럽게 녹은 종합 해석이 노출된다

### AC-5 (US-3, Sad) — LLM 호출 실패
Given 카드는 뽑혔으나 LLM 호출이 실패했을 때
When 결과 화면을 그리면
Then 룰베이스 의미만 노출되고 상단에 "온라인 복구 시 해석 업데이트" 배너가 뜬다

### AC-6 (US-4, Happy) — PWA 설치
Given 사용자가 PWA 지원 브라우저에서 앱에 접속했을 때
When 브라우저 설치 배너 또는 앱 내 안내로 홈화면 추가하면
Then 홈 아이콘으로 standalone 모드 실행되고 주소창이 보이지 않는다

### AC-7 (US-5, Edge) — 오프라인 fallback
Given 사용자가 오프라인 상태로 앱에 진입했을 때
When 질문을 던지고 카드를 뽑으면
Then 룰베이스 의미만 노출되고 "오프라인 — LLM 해석은 온라인 복구 시 적용" 표시가 보인다

### AC-8 (US-6, Happy) — 히스토리 조회
Given 과거에 1건 이상 결과를 본 사용자가
When 히스토리 탭에 진입하면
Then 날짜·카테고리·뽑힌 카드·해석 요약이 시간역순으로 리스트된다

### AC-9 (US-6, Sad) — 히스토리 비어있음
Given 결과 이력이 0건인 사용자가
When 히스토리 탭에 진입하면
Then "아직 본 결과가 없어요" 빈 상태와 첫 질문 CTA가 노출된다

### AC-10 (US-7, Happy) — 궁합 결과
Given 최애가 등록된 사용자가 궁합 카테고리에 진입하여
When 셔플 후 카드 [UNVERIFIED 권장 6장: 나 3 + 최애 3]을 뽑으면
Then "나의 에너지 / 최애의 에너지 / 둘 사이 케미" 3블록 해석이 출력된다

### AC-11 (US-8, Happy) — 마주침 결과 (실존/가상 톤 자동 분기)
Given 최애가 등록된 사용자가 마주침 카테고리에 진입하여
When 셔플 후 카드 [UNVERIFIED 권장 3장: 시기/장소/가능성]을 뽑으면
Then 3블록 해석이 출력되며 LLM이 최애를 비실존(2D·캐릭터)으로 판단하면 톤이 "상징적 만남(꿈/창작/팬아트)"으로 자동 조정된다

### AC-12 (US-9, Happy) — 영업 결과
Given 최애가 등록된 사용자가 영업 카테고리에 진입해 영업 대상(자유 텍스트)을 입력 후
When 카드를 뽑으면
Then "대상 수용성 / 최애 매력 노출 / 성공 가능성" 3블록 해석이 출력된다

### AC-13 (US-10, Happy) — 티켓팅 결과
Given 최애가 등록된 사용자가 티켓팅 카테고리에 진입해 예매 컨텍스트(공연명, 선택)를 입력 후
When 카드를 뽑으면
Then "예매일 운 / 좌석 운 / 종합 성공 가능성" 3블록 해석이 출력된다

### AC-14 (US-11, Happy) — 덕질 지속력 결과
Given 최애가 등록된 사용자가 덕질 지속력 카테고리에 진입하여
When 카드를 뽑으면
Then "현재 열정 / 다가올 권태기 신호 / 극복·재점화 카드" 3블록 해석이 출력된다

### AC-15 (US-12, Happy) — 컴백/활동운 결과 (주어가 최애)
Given 최애가 등록된 사용자가 컴백 카테고리에 진입하여
When 카드를 뽑으면
Then "차기 활동 성격 / 팬덤 반응 / 응원 포인트" 3블록이 출력되며 해석 주어가 사용자 본인이 아닌 최애로 유지된다

### AC-16 (US-13, Happy) — 입덕 결과
Given 사용자가 입덕 카테고리에 진입해 새 입덕 후보(자유 텍스트)를 입력 후
When 카드를 뽑으면
Then "지금이 적기인가 / 기존 최애와의 양립성 / 입덕 후 만족도" 3블록 해석이 출력된다

### AC-17 (US-14, Happy) — 자유 질문 → 카테고리 매핑
Given 최애가 등록된 사용자가 자유 질문 카테고리에 자연어 질문을 입력했을 때
When 시스템이 입력을 분석해 8개 카테고리 중 가장 가까운 스프레드로 매핑 후 카드를 뽑으면
Then 결과 상단에 "→ N 스프레드로 해석" 매핑 라벨이 보이고 해당 스프레드 결과가 출력된다

### AC-18 (US-14, Sad) — 매핑 불가 fallback
Given 자유 질문이 어느 카테고리에도 매핑되지 않을 때 (LLM 신뢰도 [UNVERIFIED] 임계값 미만)
When 결과 화면을 그리면
Then 기본 3장 스프레드(과거/현재/미래)로 fallback 해석되고 "일반 스프레드로 해석" 안내가 노출된다

### AC-19 (US-15, Happy) — Google 로그인 + 멀티디바이스 동기화
Given 다른 디바이스에서 같은 Google 계정으로 로그인할 때
When 인증이 성공하면
Then 기존 최애·히스토리가 Firestore에서 로드되어 즉시 노출된다

### AC-20 (US-15, Sad) — 로그인 실패 / 거부
Given 사용자가 Google OAuth 동의를 거부하거나 인증이 실패했을 때
When 로그인 화면으로 복귀하면
Then 에러 메시지가 노출되고 재시도 버튼만 제공된다 (게스트 모드 없음)

### AC-21 (US-16, Happy) — 일일 한도 내 정상 사용
Given 오늘(KST 기준) 결과 조회 N회 (N < 5)인 사용자가
When 새 결과를 요청하면
Then 카운터가 N+1 로 증가하고 결과가 정상 출력되며 잔여 횟수가 화면에 표시된다

### AC-22 (US-16, Sad) — 일일 한도 초과
Given 오늘 결과 조회 5회를 채운 사용자가
When 새 결과를 요청하면
Then 결과 화면이 그려지지 않고 "오늘 5회 모두 사용 — 내일 다시 만나요" 안내 + KST 0시 리셋 시각이 노출된다
