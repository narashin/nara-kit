# 번역투 패턴 (25~36, 40)

영어 → 한국어 직역에서 발생하는 부자연 구조. LLM은 영어 학습 데이터의 영향을 강하게 받아 번역투를 자동 생성한다.

---

## 패턴 25: 조사 번역투 "~에 대해"

- **검증**: 📊 경험적
- **심각도**: S2
- **버전**: v1.4.0

**현상**: 영어 "about/regarding"의 직역. 한국어는 "~을(를)"로 충분한 경우가 많음.

**검출 기준**:
- "~에 대해" 빈도 (문단당 2회+)
- "~에 대한"도 함께 빈출

**예시 (AI)**:
- "이 문제에 대해 논의했다."
- "사용자 피드백에 대한 분석이 필요하다."

**교정**:
- "이 문제를 논의했다."
- "사용자 피드백을 분석해야 한다."

## 패턴 29: 동사 잉여 "수행하다 / 진행하다"

- **검증**: 📊 경험적
- **심각도**: S2
- **버전**: v1.4.0

**현상**: 명사화 + 일반 동사 보조. "분석을 수행하다" = "분석하다".

**검출 기준**:
- "~을 수행하다" "~을 진행하다" 반복
- 직접 동사형이 있는데도 우회

**예시 (AI)**:
- "데이터 분석을 수행했다."
- "회의를 진행했다."

**교정**:
- "데이터를 분석했다."
- "회의했다." 또는 "회의를 열었다."

## 패턴 30: 피동 남용 "되어지다 / 지게 되다"

- **검증**: 📊 경험적
- **심각도**: S1
- **버전**: v1.4.0

**현상**: 이중 피동. 한국어 문법상 비표준. "되어지다"는 "되다"의 잘못된 강조.

**검출 기준**:
- "되어진다" "되어졌다" "지게 된다" 출현

**예시 (AI)**:
- "이 정책은 곧 시행되어질 것이다."
- "결과가 보고되어졌다."

**교정**:
- "이 정책은 곧 시행된다."
- "결과가 보고됐다."

## 패턴 32: 가능 표현 "~할 수 있다" 남발

- **검증**: 📊 경험적
- **심각도**: S2
- **버전**: v1.4.0

**현상**: 영어 "can/may"의 자동 직역. 능동형 단정으로 충분한데 가능형으로 약화.

**검출 기준**:
- 문단마다 "할 수 있다" 출현
- 단정해도 될 사실을 가능형으로 표현

**예시 (AI)**:
- "이 도구를 사용하면 시간을 절약할 수 있다."
- "성과를 측정할 수 있다."

**교정**:
- "이 도구로 시간이 줄어든다."
- "성과를 측정한다."

## 패턴 33: 미래 단정 "~것이다" 남발

- **검증**: 📊 경험적
- **심각도**: S1
- **버전**: v1.4.0

**현상**: 영어 "will"의 자동 직역. 현재형으로 충분한데 미래 추정으로 약화하거나, 단정 종결에 자동 "것이다" 붙임.

**검출 기준**:
- 문단마다 "~것이다" 종결
- 현재 사실을 굳이 "~할 것이다"로

**예시 (AI)**:
- "이 방법은 효과적일 것이다."
- "사용자는 만족할 것이다."

**교정**:
- "이 방법은 효과적이다."
- "사용자는 만족한다."

## 패턴 40: AI 결말 단언 ("~할 것이다 / ~될 것이다")

- **검증**: 📊 경험적
- **심각도**: S1
- **버전**: v1.6.0

**현상**: 패턴 33의 결말 특화 버전. 글 마지막에 미래 단언으로 마무리. "더 나은 미래가 될 것이다", "성장의 기반이 될 것이다".

**검출 기준**:
- 마지막 문장이 "~될 것이다"로 종결
- 추상적 미래 약속

**예시 (AI)**:
- "이러한 노력은 더 나은 미래의 기반이 될 것이다."

**교정**:
- "이런 노력이 다음 단계를 만든다."
- 또는 결말 단언 자체를 제거 — 본문 마지막 사실 문장으로 자연 종결
