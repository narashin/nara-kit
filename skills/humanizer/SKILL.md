# humanizer — 한국어 AI 작문 패턴 감지·교정

LLM이 생성한 한국어 텍스트의 특징적 마커(쉼표 과다, 경직된 띄어쓰기, 번역투, AI 유행어, 단조로운 구조)를 감지해 자연스러운 한국어로 윤문한다. KatFishNet 논문(ArXiv 2503.00032v4, 2025) 기반 6카테고리 40패턴.

## 적용 범위

- **대상**: 한국어 텍스트만. 영어/혼합 텍스트는 한국어 부분에만 적용
- **목표**: 자연스러움 + 의미 보존. 완벽한 문법이 아니라 인간이 쓴 듯한 흐름
- **금지**: 사실/숫자/고유명사/직접 인용 변경, 격식 수준 임의 변환, 작성자 톤 왜곡

## 검출 패턴 (6카테고리 40개)

| Cat | # | 정확도 | 대표 패턴 |
|-----|---|--------|-----------|
| 문장부호 | 7 | 94.88% AUC | 쉼표 과다, 영어식 배치, 연결어미 뒤 쉼표 |
| 띄어쓰기 | 3 | 79.51% AUC | 의존명사 경직성, 보조용언, 숫자 띄어쓰기 |
| 품사 다양성 | 3 | 82.99% AUC | 명사 과다, 동사/형용사 빈곤 |
| 어휘 | 10 | 경험적 | AI 유행어, 한자어 과다, 복수형 -들, 대명사 과다 |
| 구조 | 4 | 경험적 | 단조로운 리듬, 3박자 리스트, 접속사 과다 |
| 번역투 | 13 | 경험적 | 에 대해/통해, 가지고 있다, 되어진다, 할 수 있다 |

상세 패턴 (검출 기준 + 과학적 근거 + 예시 + 교정 전략):
- [punctuation-patterns.md](references/punctuation-patterns.md) — 1~7
- [spacing-patterns.md](references/spacing-patterns.md) — 8~10
- [pos-patterns.md](references/pos-patterns.md) — 11~13
- [vocabulary-patterns.md](references/vocabulary-patterns.md) — 14~20, 37~39

해당 카테고리 패턴이 감지될 때만 참조 파일 로드. 텍스트가 짧거나 마커가 거의 없으면 로드 스킵.

## 심각도 + 자연도 등급

**Severity**:
- **S1 (Critical)**: 1회만 나와도 결정적 AI 신호. 무조건 교정.
- **S2 (Strong)**: 1~2회 허용, 3회+ 반복 시 신호. 빈도 기반.
- **S3 (Weak)**: 단독 약함, 다른 패턴과 중첩 시 결정적.

**Naturalness Grade** (윤문 후 평가):
- **A**: S1 0 + S2 ≤ 2 — 인간 작성처럼 읽힘
- **B**: S1 1~2 또는 S2 3~5 — 자연스럽지만 사소한 흔적
- **C**: S1 3+ 또는 S2 6+ — 추가 윤문 권장
- **D**: 다중 카테고리 위반 (S1 5+ + S2 8+) 또는 변경률 가드 발동

## 작업 흐름

### Step 1 — 입력 수집

- 인자/대화 컨텍스트/파일 경로에서 한국어 텍스트 확보
- 파일이면 Read 도구로 로드
- 격식 수준 식별 (격식체 vs 반말) — 기본 보존

### Step 2 — 패턴 스캔

6카테고리 체크리스트로 체계 분석:
- 문장부호: 쉼표 빈도 / 위치 / 영어식 콜론 / 연결어미 뒤 쉼표
- 띄어쓰기: 의존명사 일관성 SD, 보조용언, 숫자 단위
- 품사: 명사 비율, 명사화(`~함/~음`) 구문
- 어휘: AI 유행어(중요하다/핵심적/효과적/지속가능한/혁신적), 한자어 과다, 복수형 -들, 대명사·지시관형사 과다, ~적 N 추상 체인, 결말 관용구·결말 단언
- 구조: 문장 길이 변동(σ), 3박자 리스트, 접속사 밀도, 경어체 균일성
- 번역투: 에 대해/통해/있어서, 가지고 있다, 되어진다/에 의해, 할 수 있다 남발, ~것이다 단언

감지 패턴이 속한 카테고리의 참조 파일만 로드.

### Step 3 — 발견 사항 정리

패턴별로:
- 텍스트 인스턴스 인용 (구체적 어절)
- 과학적/경험적 근거
- S1/S2/S3 부여 이유

### Step 4 — 재작성

- 의미·사실·고유명사 보존
- 격식 수준 유지
- 감지 패턴 전부 교정
- 작성자 목소리 보존 — 과교정 금지

### Step 4.5 — 의미 보존 자체검증 (필수 게이트)

출력 전 6항 점검. **위반 발견 시 수정 후 재점검.**

- [ ] 수치·날짜·고유명사 보존
- [ ] 인과관계 방향 유지 ("X 때문에 Y" → 반전 금지)
- [ ] 부정 표현 (있다/없다, 가능/불가능, 찬성/반대) 비반전
- [ ] 직접 인용 (큰따옴표 내) 원형 유지
- [ ] 핵심 주장·결론 유지
- [ ] 격식 수준 (격식체/반말) 일관

통과 시 출력 끝에 `✓ 의미 보존 검증: 6항 통과` 명시. 위반 수정 시 변경 내역 기록.

### Step 4.6 — 변경률 가드

원본 vs 자연스러운 버전 어절 단위 비교. 의미 단위 변경만 카운트 (단순 띄어쓰기·문장부호 제외).

| 변경률 | 동작 |
|--------|------|
| <30% | 그대로 출력 (정상) |
| 30~50% | 출력 끝에 `변경률 N%로 다소 큼. 의미 보존 재확인 권장` 한 줄 추가 |
| >50% | 출력 중단. 사용자에게 `"변경률이 N%로 과도합니다. 더 보수적으로 재작성해드릴까요?"` 확인 요청 |

근거: 과교정은 작성자 톤·뉘앙스 손실 위험. 정량 게이트로 의미 보존 강화.

### Step 5 — 결과 출력

**정상 형식**:

```
## 분석 결과

**감지된 패턴 요약**: S1 N건, S2 M건, S3 K건
**자연도 등급**: A | B | C | D — [등급 근거 한 줄]

다음 AI 작문 패턴을 감지했습니다 (심각도 순):

1. **패턴 X [S1]**: [설명]
   - [텍스트 인스턴스]
   - [과학적/경험적 근거]
   - [심각도 부여 이유]

2. **패턴 Y [S2]**: [...]

[S1 → S2 → S3 순 정렬]

---

## 자연스러운 버전

[재작성 텍스트]

---

## 주요 변경 사항 요약

- [근거와 함께 변경 1]
- [근거와 함께 변경 2]
✓ 의미 보존 검증: 6항 통과
```

**이미 자연스러운 경우**:

```
## 분석 결과

AI 마커가 거의 없습니다. 사소한 문제만 발견:

- [사소 문제 1]
- [사소 문제 2]

원본 또는 약간 수정한 버전:

[텍스트]
```

## 특수 상황

| 상황 | 처리 |
|------|------|
| 1~2문장 짧은 텍스트 | 명백한 마커(쉼표/어휘/띄어쓰기)에만 집중. 한계 명시. |
| 이미 자연스러운 텍스트 | 잘 작성됨 인정. 사소한 제안만. 과교정 금지. |
| 기술/전문 분야 | 한자어라도 기술 용어 보존. 구조·문장부호에 집중. |
| 한국어 + 영어 혼합 | 영어 부분 원형 유지. 한국어만 교정. |
| 사용자 격식 지정 | 요청대로 조정. 40패턴 교정은 동일 적용. |

## 의미 보존 원칙

- 사실·핵심 메시지 절대 변경 금지
- 작성자 의도된 톤·목적 유지
- 격식 수준 (격식체 vs 반말) 존중
- 같은 격식 범위 내에서 리듬 다양화는 허용

## 출력 contract

- 응답 = 분석 + 재작성본 (nara-kit 표준 receipt 형식 적용 안 됨 — 윤문은 산출물 자체가 응답)
- 파일 경로가 입력이면 Edit으로 inline 교체 옵션 제공 (사용자 동의 시)
- 외부 시스템 부수효과 없음
