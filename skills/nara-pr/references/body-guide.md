# PR Body Guide — 섹션 규칙 + 예시

PR 본문은 리뷰어·QA가 코드를 열지 않고 읽는 문서다. 짧은 문장, 서술형 평문. 커밋 메시지 나열·파일 경로 나열 금지.

## 섹션 구성 (순서 고정)

1. `## 요약` — 무엇을 왜 바꾸는지 2~3문장. 비개발자가 읽어도 이해되게.
2. `## 주요 변경` — 불릿 3~7개. "무엇이 어떻게 달라졌는지" 기준으로 묶는다.
3. `## 확인 방법 (QA 가이드)` — **필수, 생략 불가**. 아래 QA Guide Rules 참조.
4. `## 배포 시 주의사항` — 마이그레이션·환경변수·배포 순서·롤백 조건. 없으면 "없음" 한 줄 (섹션 자체는 생략 금지 — "없음" 판단도 정보다).
5. `## Linked Issues` — `Closes TICKET-ID`

## QA Guide Rules (확인 방법 섹션)

테스터는 이 PR 전후로 **사용자 입장에서 무엇이 달라지는지** 알아야 테스트할 수 있다.

**본체는 기대 동작 대비 표** — 계약은 동작이지 도구가 아니다. 어떤 방법으로 확인할지는 테스터가 자기 환경에 맞게 고른다:

  | 어디서 | 무엇을 하면 | 기존 동작 | 변경 후 동작 |
  |--------|------------|----------|--------------|

- "어디서"는 UI가 있으면 화면 진입 경로(메뉴 > 페이지 > 버튼), API 전용이면 메서드 + 엔드포인트.
- 새 기능이라 기존 대응물이 없으면 기존 동작 칸에 "— (기능 없음)"으로 표기.
- 겉으로 변화가 없는 변경(리팩터링·성능·내부 로직)은 "동작 변화 없음"을 명시하고, 대신 회귀 확인 포인트(깨지면 안 되는 기존 동작)를 적는다.

**재현 도구·명령은 참고 힌트로 강등** — 표 아래 "참고:" 단락에 선택 제공:

- 직접 실행해 검증하지 않은 명령(curl 등)을 유일한 절차처럼 쓰지 않는다. 싣더라도 "예:"로 표기하고 환경에 따라 다를 수 있음을 밝힌다.
- SSO/게이트웨이(Okta 등) 뒤 API는 curl용 토큰을 직접 발급받을 수 없을 수 있다 — "브라우저 로그인 → DevTools Network → Copy as cURL → URL·바디만 교체" 우회를 힌트로만 적는다.
- 스크린샷은 필수 아님 — 대부분 로그인/보안 환경이라 촬영 불가. 표의 "어디서/무엇을"이 스크린샷 없이 찾아갈 수 있을 만큼 구체적이면 충분. 촬영 가능한 공개 화면이면 첨부 권장.

## naranizer Post-Pass 상세

본문 초안 완성 후, 같이 설치된 `../naranizer/SKILL.md` (이 스킬 디렉토리 기준 상대 경로)를 Read하고 transform 플로우로 본문을 재작성. register는 ③ 기술 의견/리뷰 고정.

- 스타일 프로필 없음 또는 naranizer 미설치 → post-pass 스킵, 초안 그대로 진행. **PR 생성을 막지 않는다** (naranizer 단독 실행의 프로필 하드 게이트는 여기 적용 안 함).
- 섹션 구조·표·수치·링크·코드·`Closes` 라인은 불변 — 문장 어투만 변환.
- naranizer trailing status는 PR 본문에 넣지 않는다. 응답 receipt에 적용 여부(`naranizer ✓` / `naranizer skipped: <이유>`) 한 줄 표기.

## Example

```
# Title
PROJ-111 feat: Add order cancellation API

# Body (Korean)
## 요약
주문 취소 API를 새로 추가했어요. 이제 사용자가 결제 완료 후에도 주문을
직접 취소할 수 있고, 취소하면 환불이 자동으로 시작됩니다.

## 주요 변경
- `POST /orders/{id}/cancel` 엔드포인트 추가
- 취소 사유 저장 + 환불 트리거 연동

## 확인 방법 (QA 가이드)
| 어디서 | 무엇을 하면 | 기존 동작 | 변경 후 동작 |
|--------|------------|----------|--------------|
| 주문 상세 화면 | 결제 완료 주문에서 "취소" 클릭 | 취소 버튼 없음 | 사유 입력 후 취소 완료, 환불 대기 상태로 전환 |

## 배포 시 주의사항
- `orders` 테이블 마이그레이션 선행 필요 (`cancel_reason` 컬럼)

## Linked Issues
Closes PROJ-111
```
