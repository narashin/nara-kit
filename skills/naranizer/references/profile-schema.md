# Style Profile Schema

프로필 문서(`~/.claude/naranizer/style-profile.md`)의 필수 구조. 아래 예시는 전부 **placeholder** — 실데이터는 로컬 프로필에만 존재한다.

## 필수 섹션 7개

```markdown
# Naranizer Style Profile — <user>

> 개인 로컬 전용. repo 커밋 금지.
> 수집: <채널>, <기간>, <N>건 (top-level X + 스레드 답글 Y)
> 제외: <고정 포맷 메시지 유형>
> 생성: <YYYY-MM-DD>

## 1. 기본 문체
존댓말/반말, 문체 혼용 규칙, 짧은 수긍 답글 패턴, 요청 화법.

## 2. 어미 시그니처 (실측 빈도)
| 패턴 | 빈도 | 용도 |
|---|---|---|
| 합니다체 | NN% | 요청, 공지 |
| ~네요 | NN% | 발견, 소감 |
감탄사, 다중 문장부호, 축약어도 여기에.

## 3. 이모지 사전
| 이모지 | 용도 |
|---|---|
| :example-bow: | 요청 마무리 |
메시지당 상한, 위치(문장 끝), 유니코드 이모지 사용 여부 명시.
특정 register 전용 이모지(상태 이모지 등)는 전용임을 표시.

## 4. 유머·은유 레퍼토리
생활 은유, 밈 어휘, 괄호 셀프태클, 취소선 등 — 원문 예시와 함께.

## 5. 구조 습관
요청 공식, 지시어 습관, 링크 처리 방식, 리스트 스타일, 태그 프리픽스,
설명 구조(결론→근거→대안 등).

## 6. Register 목록
번호 + 이름 + 각 register의 문체 규칙. 고정 포맷 register는 템플릿 포함.

## 7. 하지 않는 것
음성 특징만큼 중요. 인사말 습관, 남발하지 않는 표현, 안 쓰는 문형.
```

## 작성 규칙

- 모든 빈도는 **실측치** — 추정 금지. 표본 수와 기간을 헤더에 명시
- 각 특징에 실제 원문 예시 1개 이상 (프로필은 로컬 전용이므로 원문 인용 가능)
- 섹션 7 ("하지 않는 것")은 생략 불가 — 과잉 모방 방지의 핵심 데이터
- 고정 포맷 메시지(스크럼 불릿, AI 생성 요약 등)를 문체 통계에서 제외했음을 명시
